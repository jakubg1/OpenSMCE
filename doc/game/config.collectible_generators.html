config/collectible_generators/*.json

Defines all Collectible Generators in the game.

Each collectible generator is stored in a separate file, located in config/collectible_generators/*.json. The name of the file is the same as ID of that generator.

To add a new collectible generator, simply create a new file in the aforementioned folder.

A single collectible generator is a single entry. Various entry types allow chaining or picking from multiple different entries.

All entries evaluate to a list of collectibles, and the collectible generator result is evaluated from the root entry.

Allowed collectible generator entry types

Debug information: this can be checked in CollectibleGeneratorEntry:evaluate().

collectible

Evaluates to a single collectible.

  • A single entry.
    • type: collectible
    • name: A collectible ID to be evaluated to.
    • conditions: A list of conditions. All of them must be satisfied. In other case, this entry will not generate any collectibles and return an empty list. More info below.

collectibleGenerator

Evaluates to another collectible generator result.

  • A single entry.
    • type: collectibleGenerator
    • name: A collectible generator ID to be evaluated.
    • conditions: A list of conditions. All of them must be satisfied. In other case, this entry will not generate any collectibles and return an empty list. More info below.

combine

Evaluates a list of given entries and returns a list of all collectibles generated.

  • A single entry.
    • type: combine
    • entries: A list of collectible entries to be evaluated.
    • conditions: A list of conditions. All of them must be satisfied. In other case, this entry will not generate any collectibles and return an empty list. More info below.

repeat

Evaluates a given entry a number of times and returns a list of all collectibles generated.

  • A single entry.
    • type: repeat
    • entry: A collectible entry to be evaluated.
    • count: How many times the entry should be evaluated.
    • conditions: A list of conditions. All of them must be satisfied. In other case, this entry will not generate any collectibles and return an empty list. More info below.

randomPick

Evaluates all the choices, discards empty ones, and selects one of the remaining results at random.

  • A single entry.
    • type: randomPick
    • pool: A list of choices to be picked from.
      • A single choice.
        • entry: A collectible entry to be evaluated.
        • weight: The bigger this number is, the more likely this option is to be chosen. Defaults to 1.
    • conditions: A list of conditions. All of them must be satisfied. In other case, this entry will not generate any collectibles and return an empty list. More info below.

Allowed collectible generator conditions

Debug information: this can be checked in CollectibleGeneratorEntry:checkCondition().

expression

Returns true if the provided expression returns true.

  • A single condition.
    • type: expression
    • expression: An expression to be evaluated.

colorPresent

Returns true if a given color is present on the board.

  • A single condition.
    • type: colorPresent
    • color: A color to be checked.

cmpLatestCheckpoint

Returns true if the profile latest checkpoint value is within defined bounds.

  • A single condition.
    • type: cmpLatestCheckpoint
    • min: If the checked value is lower than this value, this check will fail.
    • max: If the checked value is greater than this value, this check will fail.
    • value: If the checked value is not equal to this value, this check will fail.
