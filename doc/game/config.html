config.json

Contains base game settings. This file is placed in the root game folder.

Valid JSON structure inside this file is required for the Boot Screen to properly recognize the game. If you make a syntax error, the game will not be listed in the Boot Screen. Look in this file if your game suddenly disappears from the list!

  • The root object.
    • name: The game name to be displayed in Rich Presence and in the game window.
    • windowTitle: The window title which the engine window will have, if defined.
    • engineVersion: Specifies the version a game is for. Not really used at the moment, but mandatory.
    • nativeResolution: The native resolution of the game.
    • richPresence: Defines the Discord Rich Presence behavior.
      • enabled: Should be always true, unless you want the game to be a secret. In that case, loading a game with this flag set to false causes Rich Presence to disconnect.
      • applicationID: Optional. If defined, this will be used as an Application ID when running this game, instead of the default OpenSMCE one.
    • useUI2: Enables experimental UI2 support. The game will launch using UI2 instead of legacy UI.
